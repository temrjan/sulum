## СТРАТЕГИЯ ПОИСКА

### Приоритет извлечения:

1. **Всегда:** SYSTEM-PROMPT.md (в контексте)
2. **При новом запросе:** NAVIGATION.md
3. **При конкретной проблеме:** PROTOCOLS.md
4. **При углублении:** Соответствующий автор

### Комбинирование:

Типичный запрос может потребовать 2-3 источника:
- Навигация → выбор автора
- Автор → техники и цитаты
- Протокол → пошаговый алгоритм

### Примеры поисковых запросов:

| Запрос пользователя | Что искать в RAG |
|---------------------|------------------|
| «Не вижу смысла жить» | protocols: потеря смысла; authors: frankl, yalom |
| «Муж меня не слышит» | navigation: отношения; authors: satir, rogers, bowlby |
| «Постоянная тревога» | protocols: тревога; authors: hayes, beck, linehan |
| «Как перестать откладывать» | protocols: прокрастинация; authors: ellis, hayes |
| «Боюсь умереть» | authors: yalom, frankl |

## МЕТАДАННЫЕ ДЛЯ CHUNK'ОВ

При индексации рекомендуется добавлять метаданные:

```json
{
  "source": "authors/01-frankl.md",
  "author": "Франкл",
  "chapter": "ГЛАВА 4: ТЕХНИКА — ПАРАДОКСАЛЬНАЯ ИНТЕНЦИЯ",
  "type": "technique",
  "keywords": ["парадоксальная интенция", "тревога", "страх", "юмор"],
  "use_cases": ["тревога", "фобии", "навязчивости"]
}
```

**Типы контента (type):**
- `biography` — биография автора
- `concept` — теоретическая концепция
- `technique` — конкретная техника
- `dialogue` — пример диалога
- `quote` — цитата
- `case` — кейс/история
- `comparison` — сравнение с другими
- `indication` — показания/когда применять
